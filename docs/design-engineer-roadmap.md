# Design Engineer Upskilling Roadmap

A self-paced learning journey for designers to become confident with CLI, version control, and vibe coding with Claude Code.

**Daily time commitment:** ~30 minutes

**Format:** Self-guided tasks (no active teaching)

## The Journey: CLI → IDE → Git → Claude Code → Web → Vibe Coding

| Day | Goal |
|-----------|----------------------------------------------|
| **Day 1** | Terminal basics: navigation, creating folders and files |
| **Day 2** | Tool installation: IDE, Node.js, Claude Code |
| **Day 3** | CLI confidence: file operations, project structure |
| **Day 4** | CLI continued: searching, finding, and reading files |
| **Day 5** | IDE basics: opening projects, navigating, editing files |
| **Day 6** | IDE continued: useful shortcuts, extensions, integrated terminal |
| **Day 7** | Git basics: init, add, commit (local only) |
| **Day 8** | Git continued: viewing history, undoing mistakes |
| **Day 9** | GitHub: creating repos, connecting local to remote, push and pull |
| **Day 10** | Claude Code basics: setup, first prompt, claude.md, creating files, /context |
| **Day 11** | HTML & CSS refresher: syntax, selectors, box model |
| **Day 12** | Classic web project: HTML, CSS, JS working together, viewing in browser |
| **Day 13** | Modern web projects: intro to React, Tailwind, and why they exist |
| **Day 14** | Claude Code agents: understanding and using agentic mode |
| **Day 15** | Claude Code skills: teaching Claude reusable capabilities |
| **Day 16** | Claude Code workflows: automating repetitive tasks |
| **Day 17** | Running apps locally: npm scripts, dev servers, and localhost |
| **Day 18** | Putting it together: design → component with Claude Code |
| **Day 19+** | Practice projects with increasing complexity |
: Daily learning schedule for the upskilling roadmap

## Phase Breakdown

### Phase 1: Command Line Fundamentals (Days 1–4)
Build confidence navigating and manipulating files without a GUI.

### Phase 2: IDE Proficiency (Days 5–6)
Learn to work efficiently in a code editor.

### Phase 3: Version Control (Days 7–9)
Understand Git for tracking changes and GitHub for backup/collaboration.

### Phase 4: Claude Code Introduction (Day 10)
First hands-on experience with AI-assisted coding.

### Phase 5: Web Fundamentals (Days 11–13)
Refresh HTML/CSS knowledge, understand classic and modern project structures.

### Phase 6: Advanced Claude Code (Days 14–16)
Master agents, skills, and workflows for efficient vibe coding.

### Phase 7: Running Applications (Day 17)
Understand npm, dev servers, and viewing work locally.

### Phase 8: Putting It All Together (Day 18+)
Apply everything learned to build real components from designs.

## Notes

Each day is designed to be completable in ~30 minutes by a beginner.
Tasks are self-guided—no active teaching required.
Learners can use Claude Code for help starting from Day 10.
Final deliverable (Day 18): A working component built from a design using Claude Code.
